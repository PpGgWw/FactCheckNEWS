[질문 1] 이 프로젝트의 핵심 목적이 무엇입니까?
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

답변:
Gemini AI를 활용하여 네이버와 다음 뉴스 기사의 신뢰도와 진위 여부를 실시간으로 
분석하는 크롬 확장 프로그램입니다. 사용자가 뉴스 기사를 읽을 때 자동으로 기사를 
추출하고, AI 분석 결과를 직관적인 패널 UI로 제공하여 허위 정보나 편향된 보도를 
판별할 수 있도록 돕습니다.

[질문 3] Manifest V3를 선택한 이유는 무엇입니까?
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

답변:
Manifest V2는 2024년부터 Chrome에서 지원 중단되었습니다. V3는 보안성이 강화되고, 
Service Worker 기반으로 성능이 개선되었으며, 권한 요청이 더 명확해져 사용자 
신뢰도가 높아집니다. 또한 background.scripts 대신 service_worker를 사용하여 
메모리 효율성이 향상되었습니다.


━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
[질문 4] 왜 네이버와 다음 뉴스만 지원합니까?
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

답변:
두 플랫폼이 대한민국에서 가장 많이 사용되는 뉴스 포털이며, DOM 구조가 상대적으로 
일관성 있어 파싱이 안정적이기 때문입니다. 모듈식 파서 아키텍처(Strategy Pattern)를 
사용하여 향후 다른 뉴스 사이트(조선일보, 중앙일보 등)도 쉽게 추가할 수 있도록 
설계했습니다.


━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
[질문 5] 모듈식 파서 구조에 대해 설명해주세요.
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

답변:
각 뉴스 사이트별로 독립적인 파서 클래스(NaverNewsParser, DaumNewsParser)를 만들고, 
공통 인터페이스(canParse, extractNewsData, updateHighlightColors, 
highlightSuspiciousSentences)를 구현했습니다. content_script.js는 현재 URL을 보고 
적절한 파서를 자동 선택하여 사용합니다. 

이를 통한 장점:
  1) 코드 재사용성 향상
  2) 새 사이트 추가 시 기존 코드 수정 불필요
  3) 각 파서의 독립적 유지보수 가능

[질문 6] Gemini API를 선택한 이유는 무엇입니까?
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

답변:
  1) 무료 티어에서도 충분한 요청 한도 제공 (분당 15회)
  2) 긴 컨텍스트 처리 능력 (뉴스 기사 전체 분석 가능)
  3) JSON 형식 응답 지원으로 파싱 용이
  4) Google의 안정적인 인프라와 빠른 응답 속도
  5) OpenAI보다 저렴한 비용으로 학생 프로젝트에 적합

[질문 7] API 키는 어떻게 관리합니까?
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

답변:
Chrome Storage API(chrome.storage.local)를 사용하여 사용자 기기에 암호화되어 
저장됩니다. 설정 패널에서 입력하면 'gemini_api_key'라는 키로 저장되며, 
service_worker.js에서 API 호출 시 동적으로 불러옵니다. 키가 없을 경우 경고

[질문 8] Service Worker의 역할은 무엇입니까?
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

답변:
service_worker.js는 백그라운드에서 실행되며 주요 역할은:
  1) Content Script로부터 분석 요청 수신
  2) Gemini API에 HTTPS 요청 전송
  3) API 응답을 Content Script로 전달
  4) 타이핑 효과를 위한 단어별 스트리밍 시뮬레이션
  5) API 오류 처리 및 재시도 로직

Content Script에서는 CORS 문제로 직접 API 호출이 불가능하기 때문에 Service 
Worker를 중계자로 사용합니다.


━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
[질문 9] CORS 문제를 어떻게 해결했습니까?
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

답변:
Chrome Extension의 Service Worker는 host_permissions를 통해 CORS 제약 없이 
외부 API를 호출할 수 있습니다. manifest.json에 
"https://generativelanguage.googleapis.com/*"를 추가하여 Gemini API 엔드포인트에 
대한 권한을 얻었습니다. Content Script → Service Worker → Gemini API 구조로 
우회합니다.


━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
[질문 10] 분석 결과는 어떻게 저장됩니까?
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

답변:
Chrome Storage API를 사용하여 세 가지 데이터를 저장합니다:
  1) gemini_api_key: API 인증 키
  2) factcheck_verdicts: URL을 키로 한 분석 결과 (진위 판정, 의심 문장)
  3) newsBlocks: 분석 이력 목록 (제목, URL, 상태, 타임스탬프)

URL은 origin+pathname으로 정규화하여 쿼리 파라미터 변경에도 동일한 기사로 
인식합니다.

[질문 11] 다음 뉴스의 dmcf-ptype 속성은 무엇입니까?
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

답변:
다음(Daum)이 기사 콘텐츠 블록을 구분하기 위해 사용하는 커스텀 속성입니다. 
ptype="text"는 텍스트 단락, ptype="figure"는 이미지와 캡션을 나타냅니다. 
DaumNewsParser는 이 속성을 활용하여 본문 텍스트만 정확하게 추출하고, 광고나 
관련 기사를 제외합니다.

[질문 12] 하이라이트 색상이 변하는 원리는 무엇입니까?
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

답변:
분석 결과의 verdict 값(true/false)에 따라 CSS 배경색을 동적으로 변경합니다:
  • true (신뢰): 초록색 계열 (rgba(34, 197, 94, 0.2))
  • false (의심): 빨간색 계열 (rgba(239, 68, 68, 0.2))

currentParser.updateHighlightColors(verdict)를 호출하여 기사 본문의 모든 
하이라이트 요소에 새로운 colorScheme을 적용합니다.

[질문 13] 의심스러운 문장은 어떻게 표시됩니까?
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

답변:
Gemini API가 suspicious_sentences 배열을 반환하면, 해당 문장을 기사 본문에서 
찾아 <mark> 태그로 감싸고 'suspicious-sentence-highlight' 클래스를 추가합니다. 
마우스 호버 시 툴팁이 나타나며, 왜 의심스러운지에 대한 설명을 표시합니다. 
CSS transition으로 부드러운 애니메이션 효과를 적용했습니다.

[질문 14] 패널의 축소/확장 기능은 어떻게 구현했습니까?
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

답변:
isHistoryCollapsed 플래그와 CSS transform을 사용합니다. 축소 버튼 클릭 시:
  1) 분석 이력 목록을 숨기고 (display: none)
  2) 축소 요약 뷰(최근 3건)를 표시
  3) 패널 높이를 250px로 축소
  4) 우하단 위치로 transform 애니메이션

이를 통해 화면 공간을 절약하면서도 핵심 정보에 빠르게 접근할 수 있습니다.

[질문 15] 타이핑 효과는 어떻게 구현했습니까?
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

답변:
service_worker.js의 simulateTypingEffect() 함수에서:
  1) 전체 텍스트를 공백 기준으로 단어 배열로 분할
  2) for 루프로 단어를 하나씩 추가

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

답변:
현재는 JS 기반으로 타입 선언 없이 동작하지만, service_worker.js에서 LLM 응답을 받아올 때 한 번 파싱 및 검증 로직을 거칩니다. 만약 Gemini API의 응답 구조가 변경되면, service_worker의 extractNewsContent 함수에서 포맷을 확인하고, 예상되는 필드(진위, 의심 문장 등)가 없으면 기본값이나 오류 메시지를 반환하도록 설계되어 있습니다. 실제 운영 중에는 응답 예시를 주기적으로 모니터링해 파싱 로직을 빠르게 업데이트할 수 있습니다.

[질문 P-22] API 키가 노출되면 보안상 문제가 되지 않나요?
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

답변:
현재 버전은 로컬 학습용으로, 키가 사용자의 브라우저 로컬 저장소에만 저장됩니다. 실제 배포 버전에서는 백엔드 프록시 서버를 통해 키를 숨기고, 클라이언트에는 직접 노출하지 않습니다. 로컬 테스트 환경에서는 키 노출 위험이 크지 않으며, 실서비스에서는 별도의 인증·보안 정책을 적용할 수 있습니다.
